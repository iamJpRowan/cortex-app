# Migrate dev workflow to skills

## Goal

Replace the Beads task-tracking tool and external runner script with a fully Claude-native development workflow. Work items are organized as **Stories** that nest recursively — a story either contains child stories (when scope is too large for one PR) or tasks (when it maps to a single PR). All orchestration moves into Claude skills. The end state is that a leaf story (one with tasks, no children) can be handed to `/work`, which drives all tasks to completion and opens a PR with no external tooling required.


## Stories

### [[new-backlog-structure.story.md]]
Restructure the backlog to make stories and tasks that can be managed inside of Obsidian

### [[build-skills.story]] 
Build Claude skills based on the current agent workflow documents

## Tasks
### Remove product view scripts
- Delete `scripts/build-product-docs.js` and `scripts/build-user-docs.js`, then remove every reference to them from `package.json` (the `build:product-docs` script and any `predev` hook entries that call these scripts). 
- Update `docs/product/backlog/README.md` to remove the "Generated by `npm run build:product-docs`" footer and replace it with a brief static description.
- Clean up any other README files carrying a generated-by footer from these scripts.

### Remove Beads
- Fully excise Beads from the repo. Delete the Beads shell scripts (`scripts/bead-reset.sh`, `scripts/beads-export-for-commit.sh`), the runner daemon (`scripts/runner.sh`), `scripts/runner.log`, and the `.beads/` data directory.
- Then audit `docs/development/agents/` for any remaining `bd`, `.beads`, or `runner.sh` references and remove them. Archive or delete `docs/development/agents/runner-design.md` (documents the old Beads runner; no longer relevant).
- Confirm `package.json` has no Beads dependencies or scripts.
